# Default local layout

Resolve defaults in this order:

- GT root: `MAIN_GT_ROOT`, then `GT_TOWN_ROOT`, then `~/gt`
- Obsidian vault: `MAIN_OBSIDIAN_ROOT`, then `OBSIDIAN_VAULT`, then `~/notes/work`

The Obsidian vault is composition-based: the vault root is an unversioned directory holding one git repository per top-level folder.

```text
<gt-root>/                  # Gas Town root
<obsidian-root>/            # Obsidian vault container (not a git repo itself)
  README.md                 # map of the vault, regenerated by /concierge:setup
  Shared/
    <Name>/                 # one git repo per entry in config shared:
  User/                     # one git repo, from config user:
    Projects/
      <Project>/
        Convoys/
        Decisions/
        Notes/
    Plans/
    PR-Reviews/
    Analysis/
    Decisions/
    Operations/
```

The `shared:` map is populated from the concierge config (`$CONCIERGE_CONFIG` inline JSON, or `~/.concierge.json`). Each entry becomes a subfolder under `Shared/` whose name is the config key with its first character capitalized.

`User/` is cloned from the `user:` URL in the concierge config. All skill-managed content (project notes, plans, PR reviews, analyses) lives inside `User/`.

Rig names seed per-project folders inside `User/Projects/` when the user hasn't named a broader project theme yet.
